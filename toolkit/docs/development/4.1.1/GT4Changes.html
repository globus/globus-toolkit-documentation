Draft: 1 May 2005

Post-4.0 Interface Changes Description Justification Implications Status
1. Proxy file specification in security descriptors is now contained within the <credential> element. This change brings in consistency in how credentials are specified in security descriptor, since certificate filenames are also specified within <credential> element. Bug 2535 Security descriptors that container <proxy-file> element, that were compatible with GT 4.0 will not work anymore. The security descriptor files will need to be rewritten with this change. Code committed to trunk
2. Command line options for clients used for delegation service, globus-credential-delegate and globus-credential-refresh have been reworked to leverage the common client options in Java WS Core client package. Also, file name to store delegated EPR has been made a required option. This makes these tools consistent with the rest of the toolkit. Bug 2973. Requiring a filename for the EPR eliminates inconsistency in how the delegate and refresh clients behave and default to. Bug 3077 Options that were used for old clients will not work anymore. Code committed to trunk
3. Security descriptors have been changed to be schema compliant and validation is done during parsing. This also led to API changes in the security.descriptor package, which has also been reworked, with some API change. Schema compliance allows for validating of the descriptors against a schema using a tool prior to parsing. This lends to early and better error reporting. The descriptor package had multiple classes with functionality overlap. Reworking it helped clean up the package. Security descriptors need to be rewritten and some changes will be needed in code that programatically creates descriptors Code committed to trunk
