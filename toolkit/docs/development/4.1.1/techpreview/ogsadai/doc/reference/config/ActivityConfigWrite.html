How to Write an Activity Configuration Document

OGSA-DAI provides a number of example activity configuration documents, so it is not necessary to write one from scratch. These can be copied, then activity elements can be added or removed to configure new OGSA-DAI services. The example documents contain the standard OGSA-DAI activities supported by each type of data resource (relational, XML, file). General purpose data delivery and transformation activities are also included.

If you use one of the example activity configuration documents, it is important to ensure that correct port number is specified for communicating with your web server. If your web server is running on a port other than 8080 then all occurences of 8080 should be replaced.

The example documents are contained in OGSA-DAI/skeletons/:

  • Relational data resources:
    • activityConfigRelational.xml
  • XML data resources:
    • activityConfigXML.xml
  • File data resources:
    • activityConfigFiles.xml
