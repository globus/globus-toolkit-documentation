Activity - deliverToGDT

Activity Type: Delivery

Description

Deliver data to another OGSA-DAI service which implements the GridDataTransport portType via invocation of its GridDataTransport::PutFully or GridDataTransport::PutBlock operations.

This activity outputs data to a corresponding inputStream activity in the other service.

Connecting deliverToGDT and inputStream activities

Example

<deliverToGDT name="delivery">
  <fromLocal from="dataSource"/>
  <toGDT streamId="inputStreamID" mode="full">
    http://host:port/path/DataService
  </toGDT>
</deliverToGDT>

Specification

Element deliverToGDT:

  • Attribute name - unique name for the activity.
  • fromLocal (required) - input stream.
    • Attribute from - name of an input stream providing the data to deliver.
  • Element toGDT (required) - the URL of an OGSA-DAI service which implements the GridDataTransport portType and to which the data is to be delivered.
    • Attribute streamId (required) - the name of an inputStream activity on the destination GDS.
    • Attribute mode (required) - the data transfer mode. One of:
      • block - the data is transferred in blocks.
      • full - all the data is transferred in one go.
    • Message-level security-related attributes (optional):
      • Used to secure the data delivery.
      • If only some attributes are provided then the others will be set to their default values.
      • The client's credentials will be used if available. If not then the credentials of the host upon which the current service is running will be used.
      • Attribute authentication (required) - message-level security authentication to be used.
        • privacy (default) - Use GSI Secure Conversation with message privacy . Third-parties will be unable to read or alter the data.
        • integrity - Use GSI Secure Conversation with message integrity. Third-parties will be able to read the data but be unable to alter the data
        • signature - Use GSI XML Signature.
      • Attribute delegation - message-level security credential delegation to be used.
        • full - full delegation.
        • limited (default) - limited delegation.
        • none - no delegation.
      • Attribute authorization - message-level security authorization to be used.
        • host - host authorization.
        • self - self authorization.
        • none (default) - no authorization.
