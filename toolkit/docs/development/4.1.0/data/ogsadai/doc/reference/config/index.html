Configuration Files

The following pages give an overview of the configuration files used to expose data resources using OGSA-DAI. We have already given an overview of the data resources that OGSA-DAI supports.

In OGSA-DAI data resource exposure is done using three kinds of document:

  • Data Resource Configuration - specifying information about a data resource including its location, its metadata and its driver classes - is done using a data resource configuration document. If you like you can skip straight to:
    • How to write a relational data resource configuration document
    • How to write an XML data resource configuration document
    • How to write a file data resource configuration document
  • Activity Configuration - specifying information about the activities a client can execute. The activity configuration dictates the activities that can be specified within a client's Perform document - is done using an activity configuration document. If you like you can skip straight to:
    • How to write an activity configuration document
  • Role Mapping and Security - specifying authorisation information on the credentials which must be supplied by clients to access a data resource and how these map to database user names and passwords - is done using a role map document. If you like you can skip straight to:
    • Role Mapping and Security
    • Simple OGSA-DAI Role Map Documents
    • Encrypted OGSA-DAI Role Map Documents
    • How to Write an OGSA-DAI Role Map Document
    • How to Use an Encrypted OGSA-DAI Role Map Document
